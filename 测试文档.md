# 新卓阅汽配 - 部署后测试文档

## 测试环境
- 服务器: yaanxinzhuoyue.top
- 商城地址: https://yaanxinzhuoyue.top/mall/
- 管理后台: https://yaanxinzhuoyue.top/admin/
- 测试工具: 浏览器（Chrome/Edge）+ F12开发者工具

---

## 一、后端服务检查

### 1.1 检查后端是否启动
```bash
ps aux | grep java
```
看到 `autoparts-mall` 进程说明启动成功。

### 1.2 检查后端日志
```bash
tail -50 /opt/autoparts-mall/app.log
```
没有 ERROR 或异常堆栈说明正常。

### 1.3 直接测试API
```bash
curl http://127.0.0.1:8081/api/product/page?pageNum=1&pageSize=5
```
返回 JSON 数据说明后端正常。

---

## 二、Nginx 检查

### 2.1 确认 nginx 运行
```bash
systemctl status nginx
```
或
```bash
ps aux | grep nginx
```

### 2.2 测试 nginx 代理
```bash
curl -I https://yaanxinzhuoyue.top/api/product/page?pageNum=1
```
返回 `HTTP/2 200` 说明代理正常。返回 `502` 说明后端没启动。

## 三、浏览器测试（按顺序）

打开 Chrome/Edge，按 F12 打开开发者工具，切换到 **Console（控制台）** 标签页。

### 3.1 测试商城首页
1. 访问: `https://yaanxinzhuoyue.top/mall/`
2. 观察 Console 是否有红色报错
3. 检查点:
   - 页面能正常加载，显示商品列表
   - 轮播图正常显示
   - 分类列表正常显示
   - **Console 无 502 错误**

### 3.2 测试商品详情
1. 点击任意商品进入详情页
2. 检查点:
   - 商品图片正常显示
   - 商品名称、价格、描述正常
   - 图片不遮挡文字
   - 富文本描述正常渲染

### 3.3 测试用户登录
1. 点击"登录"按钮
2. 输入测试账号密码
3. 检查点:
   - 登录成功，右上角显示用户名
   - Console 无报错

### 3.4 测试管理后台
1. 访问: `https://yaanxinzhuoyue.top/admin/`
2. 用管理员账号登录
3. 检查点:
   - 登录成功进入仪表盘
   - 左侧菜单全部可点击
   - **Console 无 el-radio 弃用警告**

### 3.5 测试商品管理
1. 管理后台 → 商品管理
2. 点击"新增商品"
3. 上传一张图片
4. 填写商品信息，在富文本编辑器中输入描述
5. 点击保存
6. 检查点:
   - 图片上传成功（有成功提示）
   - 商品保存成功
   - 回到列表能看到新商品
   - 去商城页面能看到新商品

### 3.6 测试轮播图管理
1. 管理后台 → 轮播图管理
2. 新增/编辑轮播图
3. 检查点:
   - 图片上传成功
   - 保存后列表显示正确
   - 商城首页轮播图更新

### 3.7 测试 WebSocket（在线客服）
1. 商城页面右下角点击"在线客服"
2. 发送一条消息
3. 检查点:
   - Console 无 `WebSocket connection failed` 错误
   - 消息能发送（即使没人回复也算正常）

## 四、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 所有API返回502 | 后端没启动 | `source /etc/profile && /opt/autoparts-mall/start.sh` |
| 图片上传失败 | OSS配置不对 | 检查 `/etc/profile.d/autoparts.sh` 中OSS变量 |
| 登录失败 | JWT_SECRET变了 | 重新登录获取新token |
| 页面空白 | 前端文件没更新 | 重新解压前端zip包 |
| WebSocket报错 | nginx没加/ws/代理 | 确认nginx.conf有/ws/配置块 |
| el-radio警告 | 旧前端缓存 | Ctrl+Shift+R 强制刷新 |

## 五、快速验证命令（服务器上一次性跑）

```bash
echo "=== 1. 后端进程 ==="
ps aux | grep java | grep -v grep

echo ""
echo "=== 2. Nginx进程 ==="
ps aux | grep nginx | grep -v grep

echo ""
echo "=== 3. API测试 ==="
curl -s -o /dev/null -w "HTTP状态码: %{http_code}\n" http://127.0.0.1:8081/api/product/page?pageNum=1

echo ""
echo "=== 4. Nginx代理测试 ==="
curl -s -o /dev/null -w "HTTP状态码: %{http_code}\n" https://yaanxinzhuoyue.top/api/product/page?pageNum=1

echo ""
echo "=== 5. 前端页面测试 ==="
curl -s -o /dev/null -w "商城首页: %{http_code}\n" https://yaanxinzhuoyue.top/mall/
curl -s -o /dev/null -w "管理后台: %{http_code}\n" https://yaanxinzhuoyue.top/admin/

echo ""
echo "=== 6. 后端最新日志 ==="
tail -5 /opt/autoparts-mall/app.log
```

全部返回 `200` 就说明部署成功。
